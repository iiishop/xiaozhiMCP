# 集群模式

集群模式用于把多台设备上的本地 MCP 工具集中到一个 server 节点，再由 server 统一提供给小智。server 连接小智 endpoint，client 连接 server 的 WebSocket 入口并注册本机工具。

## 典型拓扑

```text
小智
  |
  | wss://api.xiaozhi.me/mcp/?token=...
  v
xiaozhiMCP server
  |-- 本地 server 组件工具
  |-- catalog_* 内置目录工具
  |-- logmcp_get_errors
  |-- cluster_* 集群管理工具
  |
  | ws://server:18888/agent
  |
  |-- Windows client: clipboard, windows_manager
  |-- macOS client: apple_music_macos
  `-- Linux client: local_schedule 或自定义工具
```

## 什么时候使用集群

适合使用集群的情况：

- 小智 endpoint 只希望由一台机器连接。
- 某些工具必须运行在特定桌面环境，例如 Windows 剪贴板或 macOS Apple Music。
- 多台设备需要共享一个工具入口。
- 想把高权限本地工具限制在 client 机器上运行，server 只负责路由。

不需要集群的情况：

- 所有工具都在同一台机器上运行。
- 只需要 server 组件，例如 Exa 搜索。
- 网络环境不允许 client 访问 server WebSocket 端口。

## Server 配置

server 需要配置小智 endpoint，并开启集群：

```toml
[xiaozhi]
endpoint = "wss://api.xiaozhi.me/mcp/?token=YOUR_TOKEN"
server_script = "app_server.py"

[cluster]
enabled = true
listen_host = "0.0.0.0"
listen_port = 18888
client_token = "use-a-long-random-token"

[components]
folder = "components"
```

启动：

```bash
uv run xiaozhimcp server --config server.toml
```

如果只做本机测试，可以使用：

```toml
[cluster]
enabled = true
listen_host = "127.0.0.1"
listen_port = 18888
client_token = "dev-token"
```

## Client 配置

client 只需要连接 server，不需要小智 endpoint：

```toml
[client]
server_url = "ws://SERVER_IP:18888/agent"
node_id = "office-windows-pc"
client_token = "use-a-long-random-token"

[components]
folder = "components"
```

启动：

```bash
uv run xiaozhimcp client --config client.toml
```

`client_token` 必须与 server `[cluster].client_token` 一致。`node_id` 必须唯一。

## 注册协议

client 建立 WebSocket 后，第一条消息必须是注册包：

```json
{
  "type": "register",
  "node_id": "office-windows-pc",
  "client_token": "use-a-long-random-token",
  "platform": "windows",
  "tools": [
    {
      "name": "clipboard_get",
      "description": "Read current Windows clipboard text content.",
      "input_schema": {"type": "object", "properties": {}}
    }
  ]
}
```

server 会检查：

- `type` 必须是 `register`。
- token 必须匹配。
- `node_id` 不能为空。
- 工具名不能与 server 本地工具冲突。
- 工具名不能与其他 client 工具冲突。

注册成功后返回：

```json
{"type": "register_ack", "ok": true}
```

## 心跳与重连

client 注册后每 10 秒发送心跳：

```json
{"type": "heartbeat", "node_id": "office-windows-pc"}
```

server 更新 `last_seen` 并返回：

```json
{"type": "heartbeat_ack", "ts": 1710000000.0}
```

client 断线或注册失败后会每 3 秒尝试重新连接。server 端如果检测到连接结束，会移除该 client 的工具映射，并取消等待中的远程调用。

## 远程工具调用

小智通过 server 调用：

```python
cluster_call_remote_tool(
    tool_name="clipboard_get",
    arguments_json="{}"
)
```

server 找到工具所在 client，发送：

```json
{
  "type": "invoke",
  "id": "req-1710000000000",
  "tool": "clipboard_get",
  "arguments": {}
}
```

client 调用本地组件后返回：

```json
{
  "type": "invoke_result",
  "id": "req-1710000000000",
  "success": true,
  "result": {"success": true, "content": "hello"}
}
```

server 远程调用超时时间为 30 秒。超时或 client 离线会返回失败结果。

## 集群管理工具

开启集群后，server 会注册以下工具。

列出 client：

```python
cluster_list_clients()
```

返回示例：

```json
{
  "success": true,
  "count": 1,
  "clients": [
    {
      "node_id": "office-windows-pc",
      "platform": "windows",
      "tool_count": 3,
      "last_seen": 1710000000
    }
  ]
}
```

列出某个 client 工具：

```python
cluster_list_client_tools(node_id="office-windows-pc")
```

列出全部远程工具：

```python
cluster_list_remote_tools()
```

调用远程工具：

```python
cluster_call_remote_tool(
    tool_name="windows_find_apps",
    arguments_json='{"keyword":"Chrome"}'
)
```

## 远程安装组件

server 可以通过目录工具要求 client 拉取组件：

```python
catalog_install_component_to_client(
    component_name="clipboard",
    node_id="office-windows-pc",
    mode="client_pull"
)
```

内部会调用 client 自动注册的工具：

```text
agent_install_component__office-windows-pc
```

payload 包含组件名、组件仓库、分支、安装目录和是否重启。安装成功后，client 会安排进程退出，以便重启后加载新组件。

## 工具名冲突规则

工具名必须全局唯一。server 会拒绝以下情况：

| 冲突类型 | 错误信息 | 处理方式 |
| --- | --- | --- |
| client 工具与 server 本地工具重名 | `tool name conflict with server local tool: <name>` | 修改 client 组件工具名前缀。 |
| 两个 client 工具重名 | `tool name conflict: <name>` | 修改其中一个 client 的工具名或禁用组件。 |
| 工具名为空 | `tool name missing` | 修复组件 `export_tools()`。 |

server 本地工具优先级最高。

## 安全建议

- `client_token` 使用随机长字符串，不要使用默认示例值。
- 不要把 `listen_host = "0.0.0.0"` 暴露到公网裸端口。
- 跨公网使用时建议放在 VPN、SSH 隧道、Cloudflare Tunnel 或反向代理之后。
- 只连接可信 client，因为远程工具可能操作本地系统资源。
- 只安装可信组件，因为组件会在 client 或 server 本机执行 Python 代码。

## 排障

| 现象 | 检查点 |
| --- | --- |
| `cluster_*` 工具不存在 | server 配置中 `[cluster].enabled` 是否为 `true`。 |
| client 一直重连 | server 地址是否可访问，端口是否开放，路径是否为 `/agent`。 |
| 注册返回 `unauthorized` | client token 是否与 server token 一致。 |
| 注册返回工具冲突 | 使用 `cluster_list_remote_tools()` 和 server 本地工具列表排查重名。 |
| 远程调用超时 | client 是否在线，工具是否阻塞超过 30 秒。 |
| 安装 client 组件失败 | 目标 client 是否在线，组件名是否存在，client 是否可访问 GitHub。 |
