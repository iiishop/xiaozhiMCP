# 组件系统

组件系统让 `xiaozhiMCP` 在不修改主仓库核心代码的情况下扩展 MCP 工具。主仓库负责发现、加载、注册和远程安装组件；组件仓库负责提供具体工具实现和 README 文档。

## 组件与核心功能的边界

适合作为组件的功能通常具备以下特征：

| 特征 | 示例 |
| --- | --- |
| 包装外部 API 或第三方 MCP server | `exa`、`apple_music_macos` |
| 需要平台特定系统能力 | `clipboard`、`windows_manager` |
| 是可选领域能力 | `local_schedule` |
| 有自己的额外依赖 | `pywin32`、外部 Apple Music 桥接项目 |

适合作为核心功能的是每个部署都需要的基础设施，例如目录管理、集群通信和错误记录。

## 组件目录结构

标准组件目录如下：

```text
my_component/
  __init__.py
  component.py
  README.md
```

运行时支持目录组件和单文件组件。推荐使用目录组件，方便携带 README 和私有辅助文件。

## 组件发现路径

运行时会扫描：

```text
./components
[components].folder 指定的目录
```

如果 `[components].folder = "components"`，只扫描一次。如果指定为其他目录，会同时扫描内置目录和用户目录。

```toml
[components]
folder = "user_components"
```

示例结构：

```text
components/
  local_schedule/
    component.py
user_components/
  private_notes/
    component.py
```

## 组件入口约定

运行时按顺序尝试以下约定：

1. 模块中存在 `build_component(config, full_config)` 工厂函数。
2. 模块中存在名为 `Component` 的类。
3. 模块中存在任意带 `register` 方法的类。

`config` 是组件同名 section，例如 `[my_component]`。`full_config` 是完整 TOML 配置。

## 必需接口

每个组件必须实现：

```python
def register(self, mcp) -> None:
    """Register tools with the FastMCP server using @mcp.tool()."""

def export_tools(self) -> list[dict]:
    """Return remote tool definitions for cluster registration."""
```

`register()` 用于 server 本地 FastMCP 工具注册。`export_tools()` 用于 client 向 server 注册远程工具，也会被 server 用于本地工具名冲突检查。

## 可选接口

```python
def supports_role(self, role: str) -> bool:
    """Return False to exclude this component from server or client role."""

def invoke_tool(self, tool_name: str, arguments: dict) -> dict:
    """Synchronous invocation used by client role."""

async def ainvoke_tool(self, tool_name: str, arguments: dict) -> dict:
    """Async invocation preferred by client role."""

def set_remote_invoker(self, fn) -> None:
    """Receive server-side remote invoker when cluster is enabled."""
```

server 启动时，如果组件提供 `set_remote_invoker()`，会注入 `cluster_server.invoke_remote_tool`。这让 server 组件可以在需要时调用远程 client 工具。

## 工具定义格式

`export_tools()` 返回的每个工具定义应包含名称、描述和 JSON Schema 输入定义：

```python
{
    "name": "my_component_do_something",
    "description": "Human-readable description of what the tool does.",
    "input_schema": {
        "type": "object",
        "properties": {
            "param1": {"type": "string", "description": "..."},
            "param2": {"type": "integer", "default": 10}
        },
        "required": ["param1"]
    }
}
```

工具名称必须在 server 本地工具和所有已连接 client 工具之间全局唯一。推荐使用组件名前缀，例如 `schedule_add_event`、`clipboard_get`。

## 最小组件示例

```python
class Component:
    def __init__(self, config=None, full_config=None):
        self.config = config or {}

    def supports_role(self, role: str) -> bool:
        return role in {"server", "client"}

    def register(self, mcp) -> None:
        @mcp.tool()
        def hello_echo(text: str) -> dict:
            return {"success": True, "text": text}

    def export_tools(self) -> list[dict]:
        return [
            {
                "name": "hello_echo",
                "description": "Echo text back to caller.",
                "input_schema": {
                    "type": "object",
                    "properties": {"text": {"type": "string"}},
                    "required": ["text"],
                },
            }
        ]

    def invoke_tool(self, tool_name: str, arguments: dict) -> dict:
        if tool_name != "hello_echo":
            raise RuntimeError(f"unsupported tool: {tool_name}")
        return {"success": True, "text": str(arguments.get("text", ""))}
```

## 组件目录仓库

远程组件仓库通过 `index.json` 声明可安装组件：

```json
{
  "name": "my_component",
  "description": "Short one-line description.",
  "version": "0.1.0",
  "path": "my_component",
  "entry": "component.py",
  "readme": "README.md",
  "platforms": ["Windows", "Linux", "MacOs"]
}
```

字段要求：

| 字段 | 说明 |
| --- | --- |
| `name` | 组件名，必须匹配目录名，只允许字母、数字、下划线和短横线。 |
| `description` | 一行摘要，用于目录列表和搜索结果。 |
| `version` | 组件版本。 |
| `path` | 组件目录相对仓库根目录的路径。 |
| `entry` | 入口文件名，默认 `component.py`。 |
| `readme` | README 文件名，默认 `README.md`。 |
| `platforms` | 支持平台数组，合法值为 `Windows`、`Linux`、`MacOs`。 |

组件 README 最后一行必须为：

```text
Platforms: Windows|Linux|MacOs
```

目录工具会解析该行作为平台兼容性兜底信息。

## 当前组件

| 组件 | 角色 | 平台 | 配置 |
| --- | --- | --- | --- |
| `exa` | server | Windows, Linux, MacOs | 需要 `api_key` 或 `EXA_API_KEY`。 |
| `local_schedule` | both | Windows, Linux, MacOs | 可选 `db_path`。 |
| `clipboard` | client | Windows | 无配置，依赖 `pywin32`。 |
| `windows_manager` | client | Windows | 无配置，依赖 `pywin32` 和 PowerShell。 |
| `apple_music_macos` | client | MacOs | 需要启用并配置安装目录。 |

## 安装组件

安装到 server：

```python
catalog_install_component_to_server(component_name="exa")
```

安装到 client：

```python
catalog_install_component_to_client(
    component_name="clipboard",
    node_id="office-windows-pc",
    mode="client_pull"
)
```

client 安装会调用该 client 自动注册的工具：

```text
agent_install_component__{node_id}
```

安装成功后，client 会在 1 秒后安排进程退出，由外部进程管理器或用户重启后加载新组件。

## 开发新组件流程

1. 在组件仓库创建 `my_component/`。
2. 添加空的 `__init__.py`。
3. 编写 `component.py` 并实现 `register()` 和 `export_tools()`。
4. 如组件支持 client 调用，实现 `invoke_tool()` 或 `ainvoke_tool()`。
5. 编写 README，并确保最后一行是 `Platforms: ...`。
6. 添加 `index.json` 条目。
7. 在主仓库通过 `catalog_install_component_to_server()` 或 client 安装工具测试。

## 设计建议

- 工具名使用稳定前缀，避免跨节点冲突。
- 平台相关导入应延迟到工具调用或安全降级，避免非目标平台启动失败。
- 工具返回值统一包含 `success` 和明确错误信息，便于小智理解失败原因。
- 组件配置只读取同名 section，避免隐式依赖其他组件。
- 需要持久化的数据默认写入用户目录，不要写入源码目录。
