# 快速开始

本页介绍从源码安装、初始化配置、启动 server 或 client 的最短路径。推荐先用 `uv` 在源码目录中运行，确认配置可用后再按需要打包或部署。

## 环境要求

- Python 3.11 或更高版本。
- `uv`，用于同步依赖和运行脚本。
- 可访问小智 MCP WebSocket endpoint。
- 如需安装远程组件，需要能访问 GitHub raw/API 地址。

安装 `uv` 后进入仓库目录：

```bash
cd xiaozhiMCP
uv sync
```

## 一分钟启动 server

server 是连接小智端点的主节点。它会通过 FastMCP 暴露本地工具，并可选开启集群入口。

1. 运行交互式配置向导：

```bash
uv run xiaozhimcp init
```

2. 选择 `server`。

3. 输入小智 endpoint，例如：

```text
wss://api.xiaozhi.me/mcp/?token=YOUR_TOKEN
```

4. 按需选择是否开启集群。

5. 保存配置后运行：

```bash
uv run xiaozhimcp
```

也可以显式运行：

```bash
uv run xiaozhimcp server
```

## 一分钟启动 client

client 用于把另一台机器上的本地工具接入 server。例如 server 部署在 Linux 主机，Windows client 提供剪贴板和窗口管理工具。

1. 在 client 机器上同步依赖：

```bash
uv sync
```

2. 运行配置向导：

```bash
uv run xiaozhimcp init
```

3. 选择 `client`。

4. 输入 server 集群地址：

```text
ws://SERVER_IP:18888/agent
```

5. 输入唯一 `node_id`，例如：

```text
office-windows-pc
```

6. 输入与 server `[cluster].client_token` 相同的共享 token。

7. 选择当前平台可用组件并保存。

8. 启动 client：

```bash
uv run xiaozhimcp client
```

client 会自动重连。断线后默认每 3 秒尝试重新连接。

## 手动配置方式

如果不使用交互式向导，可以复制示例配置：

```bash
cp config.example.toml config.toml
```

最小 server 配置：

```toml
[xiaozhi]
endpoint = "wss://api.xiaozhi.me/mcp/?token=YOUR_TOKEN"
server_script = "app_server.py"
```

最小 client 配置：

```toml
[client]
server_url = "ws://SERVER_IP:18888/agent"
node_id = "my-pc-01"
client_token = "YOUR_SHARED_TOKEN"
```

## 使用指定配置文件

配置文件不一定要放在当前目录或用户目录。可以通过 `--config` 指定：

```bash
uv run xiaozhimcp --config ./config.toml
uv run xiaozhimcp server --config /etc/xiaozhimcp/server.toml
uv run xiaozhimcp client --config C:\Users\me\.xiaozhimcp\client.toml
```

## 安装组件

启动 server 后，小智可以调用目录工具安装组件。也可以在初始化向导中选择组件。

列出组件：

```python
catalog_list_components()
```

搜索日程组件：

```python
catalog_search_components(query="schedule", fuzzy=True, readme=False, platform="")
```

安装到 server：

```python
catalog_install_component_to_server(component_name="local_schedule")
```

安装到指定 client，需要先开启集群且目标 client 已连接：

```python
catalog_install_component_to_client(
    component_name="clipboard",
    node_id="office-windows-pc",
    mode="client_pull"
)
```

当前阶段只支持 `client_pull`。`server_push` 会返回 `not_implemented`。

## 验证运行状态

server 启动后可以通过小智调用内置工具确认状态。

查看可用组件：

```python
catalog_list_components()
```

查看已连接 client：

```python
cluster_list_clients()
```

查看远程工具：

```python
cluster_list_remote_tools()
```

查看运行时错误：

```python
logmcp_get_errors(limit=50)
```

## 常见启动问题

| 现象 | 可能原因 | 处理方式 |
| --- | --- | --- |
| 提示 `No config.toml found` | 当前目录和 `~/.xiaozhimcp` 都没有配置。 | 运行 `xiaozhimcp init` 或复制 `config.example.toml`。 |
| client 启动时报 `client mode requires...` | `[client]` 缺少 `server_url`、`node_id` 或 `client_token`。 | 补全 client 配置。 |
| client 注册失败 `unauthorized` | client token 与 server token 不一致。 | 确保 `[client].client_token` 等于 server `[cluster].client_token`。 |
| client 注册失败 `tool name conflict` | 多个节点或本地 server 使用了同名工具。 | 修改组件工具名前缀或禁用冲突组件。 |
| 组件没有加载 | 组件目录不在 `./components` 或 `[components].folder`。 | 检查目录结构和 `component.py`。 |
