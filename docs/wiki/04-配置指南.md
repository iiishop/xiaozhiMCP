# 配置指南

`xiaozhiMCP` 使用 TOML 配置。推荐新用户运行 `xiaozhimcp init` 生成配置，熟悉字段后再手动维护。配置文件可能包含 token 和 API key，不要提交到公开仓库。

## 配置文件查找顺序

未传入 `--config` 时，CLI 按以下顺序查找：

```text
./config.toml
~/.xiaozhimcp/config.toml
```

也可以显式指定：

```bash
xiaozhimcp --config /path/to/config.toml
xiaozhimcp server --config ./server.toml
xiaozhimcp client --config ./client.toml
```

## 最小 server 配置

```toml
[xiaozhi]
endpoint = "wss://api.xiaozhi.me/mcp/?token=YOUR_TOKEN"
server_script = "app_server.py"
```

server 模式需要 `[xiaozhi].endpoint`，用于连接小智 MCP WebSocket 端点。

## 最小 client 配置

```toml
[client]
server_url = "ws://SERVER_IP:18888/agent"
node_id = "my-pc-01"
client_token = "YOUR_SHARED_TOKEN"
```

client 模式需要全部三个字段。`node_id` 必须在同一个集群内唯一。

## 完整配置示例

```toml
[xiaozhi]
endpoint = "wss://api.xiaozhi.me/mcp/?token=replace_with_your_token"
server_script = "app_server.py"

[cluster]
enabled = true
listen_host = "0.0.0.0"
listen_port = 18888
client_token = "replace_with_shared_client_token"

[client]
server_url = "ws://SERVER_IP:18888/agent"
node_id = "office-pc"
client_token = "replace_with_shared_client_token"

[catalog]
repo_url = "https://github.com/iiishop/xiaozhiMCP-components.git"
branch = "main"
timeout_seconds = 20

[components]
folder = "components"

[logmcp]
db_path = ""

[exa]
api_key = "replace_with_your_exa_api_key"
base_url = "https://api.exa.ai"

[local_schedule]
db_path = ""
```

实际部署时，server 和 client 通常使用不同配置文件。server 配置 `[xiaozhi]` 和 `[cluster]`；client 配置 `[client]` 和需要的组件 section。

## `[xiaozhi]`

| 字段 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `endpoint` | server 必填 | 无 | 小智 MCP WebSocket endpoint，通常包含 token。 |
| `server_script` | 否 | `app_server.py` | 旧版 `mcp_router.py` 使用的本地 MCP server 脚本路径。 |

示例：

```toml
[xiaozhi]
endpoint = "wss://api.xiaozhi.me/mcp/?token=YOUR_TOKEN"
server_script = "app_server.py"
```

## `[cluster]`

server 端使用。开启后允许 client 节点连接并注册远程工具。

| 字段 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `enabled` | 否 | `false` | 是否开启集群入口。支持 `true`、`1`、`yes`、`on`。 |
| `listen_host` | 否 | `0.0.0.0` | WebSocket 监听地址。 |
| `listen_port` | 否 | `18888` | WebSocket 监听端口。 |
| `client_token` | 开启集群时必填 | 无 | client 注册时使用的共享 token。 |

示例：

```toml
[cluster]
enabled = true
listen_host = "0.0.0.0"
listen_port = 18888
client_token = "use-a-long-random-token"
```

client 连接地址为：

```text
ws://SERVER_IP:18888/agent
```

## `[client]`

client 端使用。

| 字段 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `server_url` | 是 | 无 | server 集群 WebSocket 地址。 |
| `node_id` | 是 | 无 | 当前机器唯一名称。 |
| `client_token` | 是 | 无 | 必须等于 server `[cluster].client_token`。 |

示例：

```toml
[client]
server_url = "ws://192.168.1.10:18888/agent"
node_id = "living-room-mini-pc"
client_token = "use-a-long-random-token"
```

## `[catalog]`

组件目录配置。server 端的 `catalog_*` 工具和 client 端安装工具都会使用它。

| 字段 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `repo_url` | 否 | `https://github.com/iiishop/xiaozhiMCP-components.git` | 组件目录 GitHub 仓库。 |
| `branch` | 否 | `main` | 读取组件目录的分支。 |
| `timeout_seconds` | 否 | `20` | HTTP 请求超时时间，内部最小值为 5 秒。 |
| `install_folder` | 否 | `[components].folder` 或 `components` | 组件安装目标目录。 |

示例：

```toml
[catalog]
repo_url = "https://github.com/iiishop/xiaozhiMCP-components.git"
branch = "main"
timeout_seconds = 20
```

## `[components]`

控制组件扫描目录。

| 字段 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `folder` | 否 | `components` | 额外用户组件目录。 |

运行时总会先扫描 `./components`。如果 `[components].folder` 指向不同路径，会继续扫描该路径。

示例：

```toml
[components]
folder = "user_components"
```

对应目录结构：

```text
components/
  local_schedule/
    component.py
    README.md
user_components/
  my_private_tool/
    component.py
```

## `[logmcp]`

控制运行时错误记录。

| 字段 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `db_path` | 否 | 空 | 设置后持久化错误历史；为空时使用默认存储策略。 |

示例：

```toml
[logmcp]
db_path = "./data/logmcp.sqlite3"
```

查看错误：

```python
logmcp_get_errors(limit=50)
```

## 组件配置 section

每个组件读取和组件同名的 TOML section。是否需要配置取决于组件。

Exa 搜索：

```toml
[exa]
api_key = "your_exa_api_key"
base_url = "https://api.exa.ai"
```

也可以通过环境变量提供 Exa key：

```bash
export EXA_API_KEY=your_exa_api_key
```

本地日程：

```toml
[local_schedule]
db_path = ""  # 为空时使用 ~/.xiaozhi_mcp/local_schedule/local_schedule.sqlite3
```

Apple Music macOS：

```toml
[apple_music_macos]
enabled = true
install_dir = "~/.xiaozhi/applemusic-mcp"
update_on_startup = true
```

Windows 剪贴板和窗口管理组件无需配置，但需要安装平台依赖，例如 `pywin32`。

## 角色自动识别规则

`xiaozhimcp` 不带子命令时会自动识别角色：

- `[client].server_url`、`[client].node_id`、`[client].client_token` 都存在且不是占位值时，作为 client。
- 否则作为 server。

为了避免误判，生产环境建议显式启动：

```bash
xiaozhimcp server --config server.toml
xiaozhimcp client --config client.toml
```

## 配置安全建议

- 不要提交 `config.toml`。
- 为 `[cluster].client_token` 使用随机长字符串。
- server 的 `listen_host` 如果只面向本机测试，可以设置为 `127.0.0.1`。
- 如果跨公网连接集群，建议通过反向代理、VPN 或隧道提供 TLS 和访问控制。
- 第三方 API key 优先使用环境变量或受控密钥管理方式。
