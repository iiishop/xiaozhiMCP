# 项目概述

`xiaozhiMCP` 是一个面向小智的单命令 MCP 节点运行时。它把本地或远程的 MCP 工具接入小智 WebSocket 端点，并通过组件目录与组件目录仓库扩展工具能力。项目的核心目标是让用户用一个命令完成启动，用一份 `config.toml` 同时描述服务器节点、客户端节点、组件目录和集群连接。

## 适用场景

`xiaozhiMCP` 适合以下场景：

- 在一台服务器或个人电脑上运行小智可调用的 MCP 工具。
- 把 Windows、macOS 或 Linux 设备上的本地能力暴露给小智，例如剪贴板、窗口管理、本地日程、Apple Music 控制。
- 通过一台 server 节点集中连接小智，再让多台 client 节点注册本机工具，形成跨设备工具集群。
- 从 `xiaozhiMCP-components` 组件目录安装可选工具，而不把所有功能硬编码到主仓库。

## 两种运行角色

同一个 `xiaozhimcp` 命令支持两种角色。

| 角色 | 启动命令 | 主要职责 |
| --- | --- | --- |
| `server` | `xiaozhimcp server` | 连接小智端点，通过 stdio 暴露 MCP 工具，加载 server 组件，管理组件目录和集群入口。 |
| `client` | `xiaozhimcp client` | 连接 server 的 WebSocket 集群入口，把本机工具注册到 server，接受远程调用。 |

未显式指定角色时，CLI 会读取配置自动判断。只要 `[client]` 中的 `server_url`、`node_id` 和 `client_token` 都是有效值，就优先作为 client；否则默认作为 server。

## 核心能力

### 单命令启动

项目提供 `xiaozhimcp` 脚本入口：

```bash
xiaozhimcp
xiaozhimcp server
xiaozhimcp client
xiaozhimcp --config /path/to/config.toml
```

CLI 会按顺序查找：

```text
./config.toml
~/.xiaozhimcp/config.toml
```

没有找到配置时，会提示运行 `xiaozhimcp init` 或复制 `config.example.toml`。

### 交互式配置

新用户推荐使用：

```bash
xiaozhimcp init
```

该向导会依次询问角色、小智端点、集群配置、客户端连接信息和可用组件，并把配置保存到 `~/.xiaozhimcp/config.toml`。

### 组件化扩展

运行时总会扫描 `./components`，并在 `[components].folder` 与 `components` 不同时额外扫描用户配置目录。组件通过 `component.py` 暴露 `register()` 与 `export_tools()`，由运行时自动加载。

内置目录管理工具可以直接从 `xiaozhiMCP-components` 仓库发现和安装组件：

```python
catalog_list_components()
catalog_search_components(query="schedule", platform="Windows")
catalog_install_component_to_server(component_name="local_schedule")
```

### 集群模式

server 可开启 `[cluster].enabled=true`，监听 WebSocket 地址，例如：

```text
ws://SERVER_IP:18888/agent
```

client 连接后会发送注册包，包含 `node_id`、平台、共享 token 和工具清单。server 维护远程工具注册表，并通过内置工具列出客户端、列出远程工具或调用远程工具。

## 仓库组成

| 文件 | 作用 |
| --- | --- |
| `cli.py` | `xiaozhimcp` 命令入口，负责配置发现、角色自动识别和调度。 |
| `app_server.py` | 主要运行时，负责 server/client 两种角色、FastMCP 注册、组件加载、目录工具和集群工具。 |
| `cluster.py` | WebSocket 集群服务端和客户端实现。 |
| `runtime.py` | 组件发现、动态加载和实例化逻辑。 |
| `catalog_core.py` | 远程组件目录读取、搜索、描述、安装逻辑。 |
| `config_guide.py` | `xiaozhimcp init` 交互式配置向导。 |
| `mcp_router.py` | 旧版桥接入口，把小智 WebSocket 消息转发给本地 stdio MCP 进程。 |
| `error_store.py` | 运行时错误记录，支持 `logmcp_get_errors()`。 |
| `config.example.toml` | 完整配置参考。 |

## 依赖与技术栈

项目要求 Python 3.11 及以上。核心依赖如下：

```toml
dependencies = [
  "fastmcp>=2.0.0",
  "requests>=2.32.0",
  "websockets>=12.0",
]
```

开发依赖包括 `pytest` 和 `ruff`。推荐使用 `uv` 安装和运行。

## 和组件仓库的关系

主仓库只保留基础运行时能力。可选工具放在 `xiaozhiMCP-components` 仓库，由 `catalog_core.py` 按 `index.json` 读取。当前已知组件包括：

| 组件 | 平台 | 用途 |
| --- | --- | --- |
| `exa` | Windows, Linux, MacOs | Exa AI 网络搜索。 |
| `local_schedule` | Windows, Linux, MacOs | 本地 SQLite 日程管理。 |
| `clipboard` | Windows | 剪贴板读写，支持 HTML 富文本。 |
| `windows_manager` | Windows | 窗口枚举、查找、聚焦、关闭和性能查看。 |
| `apple_music_macos` | MacOs | macOS Apple Music MCP 桥接。 |

## 安全注意事项

- `config.toml` 会包含小智 endpoint token、集群共享 token、第三方 API key，不要提交到 Git。
- 集群 client token 是 server 和 client 的共享密钥，应使用随机长字符串。
- client 暴露的是本机能力，建议只连接可信 server。
- 工具名称在 server 和所有 client 间必须全局唯一，避免远程调用歧义。
